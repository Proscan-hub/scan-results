Methodology & Scoring

Analysis Approach

Static Application Security Testing (SAST) analyzes source code, bytecode, and binary code for security vulnerabilities without executing the application. Pattern matching, taint analysis, and data-flow analysis identify potential weaknesses at the code level.

Risk score (0–100) is computed from weighted severity counts, exploit maturity, EPSS probability, affected surface area, and patch availability. Higher scores indicate greater risk.
